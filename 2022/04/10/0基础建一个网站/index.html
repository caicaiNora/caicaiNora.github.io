#首先，先捋一下整体思路

网站从我们的电脑上建立，通过一些看不见的通道，传向万维网上。你可以把你的网站文件看作一个包裹，我们打包好送上路，他们就沿着我们规定好的路线传向公开网络世界。
所以，建立一个网站，我们要做什么呢：建立电脑上的仓库（网站相关内容的文件夹），在万维网上建一个空间站、仓库（储存我们的文件），指一个传输的道路，传输文件。

1.建立电脑上的仓库

为了保证这个仓库的顺畅运行，以及让我们的电脑进入仓库建立流程，我们需要下载几个软件。

1.1 node.js

下载地址：https://nodejs.org/en/

检查安装效果：键盘按一下Win+R打开cmd（输入命令符的一个面板）

输入以下命令检查安装的软件的版本:
node -v　查看node版本
npm -v　查看npm版本
返回这样的数据就没问题～

1.2.Git

下载地址：https://git-scm.com/download/win　
一路next吧，除了这一步

输入以下命令检查Git版本：

node -v
双横线！version不缩写！

1.3.Hexo

重点来了啊，我们的博客是用Hexo建立的，并且之后对网站做修改，也会在按照Hexo的文件夹内进行，所以请自己建一个保存网站内容的文件夹，并打开文件夹，我们是打开了，电脑不知道呀，怎么通知它呢，请在文件夹空白处右键，此时你的命令行里会出现“Git Bash here”,点击它，就会弹出这一个Git窗口，Git就是一个使者，告诉你的电脑你要做什么。观察一下这个冒号前面的文件层级。是不是到了你的博客文件夹，是的话，那就是通知成功了。

输入命令，安装Hexo
npm install -g hexo-cli
依旧查看一下版本
hexo -v
初始化Hexo
hexo init （此处填你的博客名，随便啥名）　
eg.hexo init blueblunora
到了这一步结束，你的本地博客文件保存仓库建立，并且做好了安装安装准备
cd （此处填你的博客名） 进入这个博客文件夹
npm install

运行之后，这个博客文件夹内出现很多文件
node_modules: 依赖包
public：存放生成的页面
scaffolds：生成文章的一些模板
source：用来存放你的文章
themes：主题
_config.yml: 博客的配置文件

更新待上传文档的内容
hexo g
开启本地预览，可以在上传之前预览页面效果
hexo s
浏览器地址栏输入 http://localhost:4000/ 即可本地预览博客

2.建一个空间站

这个仓库，我们会建立在一个叫GitHub的网站上建立。
那么，先注册：https://github.com/
新建一个仓库：右上角头像-“Your repositories”-绿色“New”按钮
Create a new repository：Repository name这里填“xxxx.github.io”-Create repository

3.指定一个传输通道

这个传输通道由一副钥匙指定方向，这个钥匙叫做SSH，当本地的钥匙和网络上的钥匙配对后，你的网页文件就被送到指定的仓库。
在继续在博客文件夹中，git bash here

git config --global user.name "yourname"
yourname为GitHub用户名
git config --global user.email "youremail"
youremail为GitHub邮箱

设置完成之后，检查一下网络有没有记录正确
git config user.name
检查钥匙的用户名
git config user.email
检查钥匙的用户邮箱

sh-keygen -t rsa -C "youremail"
一路回车，直到这个界面出现

在电脑中，找到.ssh这个文件夹，里面包含了你的钥匙密码
ssh，简单来讲，就是一个秘钥，其中，id_rsa是你这台电脑的私人秘钥，不能给别人看的，id_rsa.pub是公共秘钥，可以随便给别人看。把这个公钥放在GitHub上，这样当你链接GitHub自己的账户时，它就会根据公钥匹配你的私钥，当能够相互匹配时，才能够顺利的通过git上传你的文件到GitHub上。

在github中设置秘钥信息，这个钥匙的密码就你知，github知，网络知。
github点击头像-settings-左侧边栏找“SSH and GPG keys”,”Add SSH keys”,title随便写，保险起见写github的用户名，避免链接出错
git bash here
ssh -T git@github.com

4.文件从本地传输到网络

在本地文件中找到 config.yml，在里面修改一些内容，不出意外，这段内容在文件最后一段
deploy:
type: git
repo: https://github.com/YourgithubName/YourgithubName.github.io.git
branch: main

安装deploy-git，使你的电脑具备部署内容到github的功能
npm install hexo-deployer-git --save
hexo clean 清空装满上传文件的文件夹
hexo g 更新待上传的文件
hexo d 部署本地文件到github

部署后，去github查看动态：去我们的repository，点击action这个标签页，一是看是否部署成功，二是看出错的话卡在哪个环节。

部署成功之后，可以在仓库的settings这里找到pages部分，查看你的网站地址，这个地址和前面的local有区别，这次的地址是网络朋友都能看到的
